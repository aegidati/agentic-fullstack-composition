# Agentic Fullstack Composition

## Purpose

This starter provides the local fullstack runtime composition foundation for Agentic platform projects.

Its role is to orchestrate multiple platform modules together in a deterministic local execution environment.

---

## Install target

This starter installs into:

app/composition

---

## Included

- Local composition foundation
- Service orchestration baseline
- Deterministic runtime composition location
- Fullstack local integration entrypoint

---

## Not included

This starter intentionally does not include:

- Production orchestration
- Cloud deployment configuration
- Business-specific service topology
- Feature-specific runtime customization

Those concerns are handled by other modules or project-specific decisions.

---

## Prerequisites

Typical runtime prerequisites:

- Docker
- Docker Compose or equivalent local runtime tooling

---

## Expected structure after installation

app/composition

---

## Installation

1. Create a project using AGENTIC-TEMPLATE.
2. Install this starter into:

app/composition

3. Configure service composition references as needed.
4. Run local composition validation checks.

---

## Post-install validation

Verify that:

- composition directory exists
- composition files are present
- compose configuration is valid
- local startup command is valid

---

## Compatibility

Compatible with:

- agentic-clean-backend
- agentic-react-spa
- agentic-flutter-client
- agentic-api-contracts-api
- agentic-postgres-dev

---

## Exit criteria

Installation is successful when:

- app/composition exists
- composition files are present
- compose configuration is valid
- startup command is valid
- no unresolved placeholders remain

---

## Notes

This starter is focused on local orchestration and developer experience.

Production deployment orchestration and environment-specific release concerns are intentionally out of scope.
